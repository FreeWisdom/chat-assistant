## 当前产品能力

机器人仅监听白名单中的：

- `项目研究`
- `每日饮食打卡🍽️`

支持三种触发方式：

- `#举手 问题`
- `@机器人 问题`
- 引用机器人的消息后继续提问

当前回答链路：

```text
微信群消息
→ 触发和群白名单校验
→ DeepSeek 判断回答路径
├─ 稳定通用问题：DeepSeek 直接回答
├─ 群业务问题：阿里云百炼知识库检索
├─ 最新外部事实：火山引擎或 Tavily 联网搜索
└─ 信息不足：追问一个关键问题
→ DeepSeek 统一润色
→ 回复原微信群
→ 验证消息是否实际发送成功，失败则重试
```

## 本分支已经完成

### 1. 云知识库替换

新增 `AliyunBailianKnowledgeClient`，支持：

- 官方 `RetrieveRequest` 和 `retrieve_with_options` 调用。
- 按群传入绑定的云知识库。
- 稠密检索和稀疏检索候选数配置。
- 百炼重排序。
- 最低相关度过滤。
- 多个云知识库结果合并和排序。
- 超时控制与失败重试。
- 将百炼 `Text`、`Score`、`Metadata` 转换为统一回答上下文。

### 2. 云端文档上传和建库

新增管理端完整流程：

```text
页面选择文档
→ ApplyFileUploadLease
→ 按百炼返回的 URL/Method/Headers 上传二进制文件
→ AddFile
→ DescribeFile 等待解析完成
→ CreateIndex + SubmitIndexJob
  或 SubmitIndexAddDocumentsJob 追加到已有知识库
→ GetIndexJobStatus 查询索引状态
→ 自动保存 Index ID、Job ID、文档 ID 和任务状态
```

页面支持 PDF、Word、PPT、Excel、WPS、Markdown、TXT、EPUB 和 MOBI。
文档生命周期信息保存在 `runtime/bot_state.db`：

- 一个逻辑文档对应多个版本。
- 当前有效版本、处理中版本和历史版本分别记录。
- 云端文件 ID 与任务 ID 不返回用户页面。
- 删除采用本地软删除保留操作历史，云端内容从检索中永久移除。

### 3. 删除本地知识库工程

已删除：

- 本地 `.md`、`.txt`、`.json` 知识文件。
- `knowledge-data/` 中的示例资料。
- 本地文件连接器。
- 网页知识库抓取连接器。
- 文档切片逻辑。
- 关键词检索器。
- Chroma 和 SentenceTransformer 向量检索。
- 旧的本地知识文件上传 API。
- 旧的本地知识文件管理界面。
- 向量索引运行目录及相关依赖、配置。

### 4. 配置和管理端

知识库配置现在只保存非敏感信息：

```yaml
knowledgeBases:
  - id: fuye-projects
    provider: aliyun_bailian
    workspaceId: 实际业务空间ID
    indexId: 页面建库后自动生成
    indexJobId: 最近一次索引任务ID
    indexStatus: PENDING
    documentIds: [已上传文档ID]
```

AccessKey 只允许放在 `backend/.env` 或系统环境变量：

```ini
ALIBABA_CLOUD_ACCESS_KEY_ID=
ALIBABA_CLOUD_ACCESS_KEY_SECRET=
ALIYUN_BAILIAN_ENDPOINT=bailian.cn-beijing.aliyuncs.com
```

## 下一步工作重心

### P0：打通页面建库和真实百炼检索

这是当前唯一的首要任务。

1. 创建具备百炼文件、知识库和检索权限的 RAM 凭证，并加入对应业务空间。
2. 填写 `backend/.env` 中的 AccessKey。
3. 在管理页分别新建“项目研究”和“饮食打卡”知识库配置。
4. 填写真实 `Workspace ID`，上传各自文档并提交建库。
5. 刷新索引任务状态，确认两个任务均为 `COMPLETED`。
6. 将两个知识库绑定到各自群，保存配置并重启机器人。
7. 分别执行项目类和饮食类问题的真实 API 冒烟测试。

完成标准：

- 两个知识库都能返回相关内容。
- 页面生成并保存正确的 `Index ID`，追加文档不会新建重复知识库。
- 返回结果分数、文档名和正文能够被机器人正确解析。
- 无结果、超时、权限错误时不会编造知识库答案。

### P1：建立检索质量基线

为两个群各准备至少 10 个测试问题，覆盖：

- 明确命中。
- 换一种说法仍能命中。
- 多文档冲突。
- 不应命中的无关问题。
- 知识库没有答案。
- 需要联网搜索的最新问题。

根据结果调整：

- `RETRIEVAL_TOP_K`
- `KNOWLEDGE_RETRIEVAL_MIN_SCORE`
- 百炼切片和索引配置
- 路由示例和标签

### P2：两群真机回归

先设置：

```ini
DRY_RUN=true
LISTEN_GROUPS=项目研究,每日饮食打卡🍽️
```

确认监听、路由和拟回复正确后，再关闭 `DRY_RUN`，逐项验证：

- 两个群各自监听。
- 三种触发方式。
- 普通问题直接回答。
- 私有业务问题命中各自云知识库。
- 最新问题走联网搜索。
- 两个群不串回复、不串知识库。
- 回复失败能够重试且不会重复发送。

### P3：提交和交付

2. 检查新增行中没有密钥。

## 暂时不要扩展的内容

在真实检索闭环稳定前，暂时不要优先做：

- 接入更多云知识库厂商。
- 长期记忆。
- 图片或语音问答。
- 自动学习群聊天内容。
- 更复杂的管理后台。
- 大范围提示词优化。

这些功能不会解决当前最关键的问题：**机器人是否能稳定地从真实百炼知识库取得正确资料并回复到正确的群。**
