# 微信群知识库机器人 MVP 收敛计划

> 目标：用最短时间得到一个能在单个真实微信群稳定试运行的 MVP。
>
> 2026-06-18 实施状态：结构重构已完成。微信层已统一替换为固定提交的
> `FreeWisdom/wxauto-4.0`，旧自写轮询、切群、内容指纹去重实现已删除。
> 当前代码结构和启动要求以 `ai-ta-bot/README.md` 为准。

## MVP 最终效果

群成员发送：

```text
#举手 这个项目适合新手做吗？
```

机器人应当：

1. 只响应 `#举手`，不误触发普通聊天。
2. 根据当前群绑定的本地知识库检索。
3. 有可靠资料时回答；资料不足时明确追问，不拿无关资料硬答。
4. 按“群 + 发送者”读取该用户最近的历史问答。
5. 记录已处理消息，重启后不重复回复。
6. 通过 DeepSeek 组织成简短微信回复。

## 最短实施顺序

### P0：当天完成核心正确性

- 只监听一个测试群。
- 只保留 `#举手` 触发。
- 修复检索“任何问题都算命中”的错误。
- 默认关闭向量检索，使用轻量关键词/FAQ 检索。
- 增加 LLM 超时和 API Key 启动校验。

完成标志：无关问题不引用副业资料；普通群消息不触发。

### P1：完成可持续运行

- 新增 SQLite。
- 按“群 + 发送者”保存最近问答。
- 持久化消息去重记录。
- 使用 wxauto4 可用的消息标识；无标识时加入时间窗口，避免永久屏蔽重复问题。

完成标志：重启后用户上下文仍在，同一条消息不重复回复，但用户以后可以再次问相同问题。

### P2：真实群灰度

- 第一轮 `DRY_RUN=true`，只记录拟回复。
- 人工检查 10 个问题。
- 第二轮只对一个白名单群开启发送。
- 连续运行 2 小时，检查漏回、误回、重复回复和群名切换。

完成标志：10 个样例全部符合预期，连续运行无重复发送。

## 暂不做

- 网页知识库抓取。
- 多知识库复杂路由。
- SaaS 平台同步。
- 在线 Markdown 编辑器。
- 用户画像自动抽取和向量化长期记忆。
- 多群并发和无人值守自动恢复。

## 预计投入

- 可进入 `DRY_RUN` 验收：约 1 个工作日。
- 可进入单群真实发送灰度：约 2 个工作日。
- 前提：继续使用当前已验证的本地 wxauto4 和微信 4.x 环境。
