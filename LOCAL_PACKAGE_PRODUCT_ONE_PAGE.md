# 本地微信群知识库助手产品目标（一页版）

## 一句话

交付一个 Windows 本地运行包：用户打开后启动本地管理服务，在网页里完成机器人、知识库、监听群、触发词和密钥配置，并能一键启动或停止微信群监听回复。

## 目标用户

需要把私有资料接入微信群问答的个人、社群运营者或小团队。他们不想改代码，也不应该直接接触 Workspace、Index、Job、File ID 这类平台内部字段。

## MVP 必须跑通的闭环

1. 用户解压本地包，双击 `启动管理台.cmd`。
2. 系统启动本地 FastAPI 服务并自动打开 `http://127.0.0.1:8000`。
3. 用户在网页中配置 DeepSeek/FastGPT、机器人身份、回复风格、微信群名、触发词和知识库。
4. 用户在网页中上传、查看、替换、删除知识库文档。
5. 用户点击“启动监听”，机器人只监听白名单群，并按触发规则回复。
6. 用户点击“停止监听”，机器人停止读取和回复微信消息。
7. 页面能看到运行状态、最近日志、配置校验结果和安全提示。

## 当前仓库已有基础

- 本地管理服务：`backend/src/ai_ta_bot/admin_app.py`
- 网页配置台：`admin-ui/src/App.jsx`
- 配置读写与备份：`backend/src/ai_ta_bot/config_store.py`
- 知识库文档上传、列表、替换、删除记录：`backend/src/ai_ta_bot/admin/routers/knowledge.py`
- 机器人启动入口：`backend/src/ai_ta_bot/__main__.py`
- 启动/重启进程雏形：`backend/src/ai_ta_bot/admin/routers/runtime.py`

## 现在最大缺口

- FastGPT 还没有真正替换百炼。
- 网页只有“重启脚本”，没有完整的一键启动/停止/状态/日志闭环。
- 配置还没有全部网页化，尤其 `.env` 密钥、监听白名单、安全开关和 FastGPT 连接测试。
- 还没有面向用户的 Windows 本地包启动器、依赖检查、首次初始化和升级策略。

## 推荐开发顺序

1. 先做运行控制闭环：启动管理台、启动监听、停止监听、状态和日志。
2. 再做配置闭环：网页配置 `.env` 安全字段、群监听、机器人、知识库、触发词。
3. 再接 FastGPT：检索、上传、替换、删除和文档生命周期。
4. 最后做打包交付：Windows zip/安装包、启动器、自检、升级和回滚。

阶段 1 和阶段 2 严格先做；阶段 3 的 FastGPT adapter 可以和阶段 2 后半段并行，但必须共用同一套配置字段、密钥隐藏规则和 provider 分派接口。最终用户交付包采用随包 Python 运行时，不要求用户提前安装系统 Python。

## MVP 不做

- 不做 Windows Service，因为微信桌面自动化需要交互式桌面会话。
- 不做多租户 SaaS。
- 不承诺个人微信官方接口级稳定。
- 不做完全离线大模型，除非后续另立本地模型版本。
